Welcome to the live demonstration walkthrough of the LoanShield lending engine.
Use this script for presentation and code review narration.

---

[ 0:00 - Introduction & Project Goals ]
"Hello everyone. Today, we're demonstrating LoanShield, a secure, compliance-first automated credit decision engine. Built on Google's ADK 2.0 framework and LangGraph, LoanShield integrates advanced PII masking, stdio-based MCP server database connections, and a multi-factor risk scoring algorithm to decide loan applications. Our primary goal is to safely, transparently, and securely automate the underwriting lifecycle."

---

[ 0:45 - Codebase & Architecture Walkthrough ]
"Let's look at the codebase. We have a strictly typed structure under `loanshield/`:
- `app/agent.py` contains our LangGraph workflow. It orchestrates gatekeeping, parallel analysis, risk scoring, and notification nodes.
- `app/mcp_server.py` implements stdio MCP servers exposing mock credit, banking, and employment databases.
- `app/skills/` contains our modular mathematical scorers and PII redactor.
- `tests/` houses our comprehensive unit, integration, and E2E graph tests."

---

[ 1:30 - Running Automated Testing ]
"Before running the live service, let's verify our engine's correctness. We can run our test suite using `make test`. This executes:
1. Unit tests checking our scoring math and PII regex masks.
2. MCP server mock integrations.
3. Graph integration tests processing all 54 applicant scenarios in our benchmark dataset.
Let's execute it:
$ make test
As you can see, all 16 test suites pass successfully, verifying that every single applicant in the benchmark dataset routes to the exact correct decision boundary."

---

[ 2:30 - Live Playground & HITL Interruption ]
"Now, let's launch our local interactive playground using `make playground`.
This starts the ADK web development server.
Let's input a loan application for Harper Robinson. Harper has thin credit, so the scoring engine calculates a risk score of 64.1 and automatically escalates the file.
Notice that the workflow pauses and emits a human underwriter checkpoint:
`Application in thin-credit zone (Score: 64.1). Approve or Reject.`
As an underwriter, we inspect the file, enter 'APPROVE' into the override input, and click resume. The workflow proceeds, finalized as AUTO_APPROVE."

---

[ 3:45 - Regulatory ECOA adverse Action notice ]
"For rejected files, our Explanation Agent automatically drafts a formal credit decision letter. Because it works in our PII safe zone, it references only masked values (like '[REDACTED_SSN]') and clearly cites the specific rejection reasons in alignment with Section 701(a) of the Equal Credit Opportunity Act. This guarantees bank audit preparedness."

"In summary, LoanShield provides a modular, secure, and fully auditable framework for automated lending. Thank you for your time."
